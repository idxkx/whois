# Change: 基于行输入的域名检索编排

## Why
运营需要能够把来自不同渠道的一批候选文本粘贴进系统，并自动组合配置里的域名后缀进行批量查询，以提升人工甄别效率并减少重复输入。

## What Changes
- 提供一个独立的行解析器，将文本列表按行切分，过滤空行后输出基础域名片段。
- 通过检索配置加载需要查询的域名后缀清单。
- 对基础片段和后缀做笛卡尔组合，逐条调用 whois 查询接口，仅需要判断是否已被注册。
- 暴露该能力供后续 agent 等功能复用，确保错误能反馈给调用方。

## Impact
- Affected specs: domain-query
- Affected code: 域名查询编排模块
